Methodology
Data definitions, metric calculations, and analytical approach

Data Source & Scope

  • Overview, Channel & Time sections: generated sample data (mock transaction events).
  • User Monitoring: data/user_monitoring_summary.csv fetched from repository root.
  • Period covered: Q1 2025 (January – March). Channels: WhatsApp, SMS, Email, Call/IVR.
  • Fallback sample data used if CSV cannot be fetched (e.g. local preview without server).

Key Metric Definitions

  • Billable Amount: Sum of billable_amount where is_charge = 1.
  • Success Rate: Proportion of records with success_flag = 1.
  • Charge Rate: Proportion of records with is_charge = 1.

User Risk Score Logic

Contextual Churn Risk Score is a composite metric: usage decline %, billable decline %, inactive days, and channel diversity. Segments: Red Alert (≥70), At Risk (40–69), Watchlist (20–39), Stable (<20). Pre-computed in the user monitoring summary CSV.

User Risk vs Value Map

X-axis: usage decline %. Y-axis: total billable amount. Bubble size = transaction volume. Color = contextual risk segment. Higher X = more usage decline risk; higher Y = higher business value to OCA.

Heatmap Construction

Aggregated by day-of-week × hour from sample transaction data. Intensity normalized relative to peak slot. Full red opacity = peak traffic window.

Limitations & Disclaimers

  • All monetary values adjusted from original data for educational privacy.
  • User names are anonymized. This dashboard is for portfolio/internship demonstration only.
  • Data does not reflect actual OCA business performance.
