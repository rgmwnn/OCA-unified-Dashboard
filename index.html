Methodology
Data definitions, metric calculations, and analytical approach

Data Source & Scope

  • Dataset: data/dashboard_unified_transactions.csv loaded from the repository.
  • Period covered: Q1 2025 (January – March).
  • Channels monitored: WhatsApp, SMS, Email, Call/IVR.
  • Each row = one transaction event.
  • Fallback sample data used if CSV cannot be fetched.

Key Metric Definitions

  • Transaction Count: Total rows (one per message/call event).
  • Message Units: Sum of message_units column.
  • Billable Amount: Sum of billable_amount where is_charge = 1.
  • Success Rate: Proportion of records with success_flag = 1.
  • Charge Rate: Proportion of records with is_charge = 1.

Channel Role Matrix Logic

Each channel is plotted by its transaction share (X-axis) vs billable amount share (Y-axis). Bubble size encodes the success/answered rate. Quadrant thresholds are set at 25% for both axes to separate high/low regions.

User Dependency Map Logic

Each user is plotted by dominant channel share (X-axis) vs total transaction volume (Y-axis). Users with >85% concentration on one channel are flagged as mono-channel. Bubble size encodes billable amount.

Heatmap Construction

Traffic is aggregated by day-of-week and hour. Intensity is normalized relative to the peak slot (max = 100). Colors use a linear red scale where full opacity = peak intensity.

Limitations & Disclaimers

  • All monetary values have been adjusted from original data for educational privacy.
  • User names are anonymized (Client A, B, C, …).
  • This dashboard is for portfolio and internship demonstration only.
  • Data does not reflect actual OCA business performance.
