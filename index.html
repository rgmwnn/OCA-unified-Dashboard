Methodology
Data definitions, metric calculations, and analytical approach

Data Source & Scope

  • Executive Overview uses data/executive_overview_summary.csv.
  • Channel Performance uses data/channel_performance_summary.csv.
  • Time Pattern Monitoring uses data/time_pattern_summary.csv.
  • User Monitoring uses data/user_monitoring_summary.csv.
  • All files are generated from the same clean SQL base for consistency.

Key Metric Definitions

  • Billable Amount: sum of billable_amount from charged transactions.
  • Success Rate: success_count / total_transactions.
  • Charge Rate: charged_transactions / total_transactions.

User Risk Score Logic

Contextual Churn Risk Score is pre-computed in SQL from usage change, billable change, inactivity, delivery friction, and high-value impact. Segments: Red Alert (≥70), At Risk (40–69), Watchlist (20–39), Stable (<20).

User Risk vs Value Map

X-axis: usage change rate. Negative values mean decline, positive values mean increase. Y-axis: total billable amount. Bubble size = transaction volume. Color = contextual risk segment.

Heatmap Construction

Aggregated by day-of-week × hour from time_pattern_summary.csv. Intensity is normalized relative to the peak slot.

Limitations & Disclaimers

  • All monetary values are adjusted from original data for educational privacy.
  • User names are anonymized. This dashboard is for portfolio/internship demonstration only.
  • Data does not reflect actual OCA business performance.
